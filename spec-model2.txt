## 1. Overview

The Quick Settings panel is a full-screen overlay surface (`ShellSurface::Overlay`)
anchored to the top-right of the screen. It serves as the **primary control hub**
for all frequently used system settings, hardware toggles, media playback,
and third-party quick actions ‚Äî accessible in a single gesture or click from
the status bar.

### Design Principles

| Principle | Description |
|---|---|
| **One-tap useful** | Every tile either toggles state or reveals a sub-page ‚Äî no dead clicks |
| **Android grid + macOS polish** | 2-column tile grid with rounded Material You containers, macOS-level animation smoothness |
| **Freedesktop-native** | All system integrations via standard D-Bus interfaces ‚Äî no proprietary daemons |
| **Context-aware** | Tiles auto-show/hide based on hardware capability (e.g. no battery tile on desktops) |
| **Keyboard-first optional** | Full keyboard navigation + shortcut to open (`Super+A` or configurable) |

### 3.6 Third-Party Tiles (StatusNotifierItem) ‚Äî **NEW**

The **StatusNotifierItem (SNI)** specification is the freedesktop.org standard
for system tray items. Chi Shell should act as a `StatusNotifierHost` to
display third-party app indicators as tiles in Quick Settings.

| Property | Value |
|---|---|
| D-Bus interface | `org.freedesktop.StatusNotifierWatcher` (session bus) |
| Discovery | Register as `org.freedesktop.StatusNotifierHost`, listen for `StatusNotifierItemRegistered` signals |
| Per-item interface | `org.kde.StatusNotifierItem` on the item's bus name |
| Item properties | `IconName`, `Title`, `Status` (`Active`/`Passive`/`NeedsAttention`), `Menu` (via `com.canonical.dbusmenu`) |

**Visual:**
- Each SNI item gets a compact tile (icon only) in a dedicated row below the
  built-in tiles, separated by a subtle divider
- Items with `NeedsAttention` status get a badge dot
- Tap ‚Üí invoke `Activate()` method on the item
- Long-press ‚Üí show the item's `dbusmenu` as a popup context menu

**Implementation approach:**
- New C++ class: `StatusNotifierWatcher` ‚Äî registers as host, tracks items
- New C++ class: `StatusNotifierItemModel` ‚Äî QAbstractListModel exposing icon,
  title, status for each registered item
- QML renders them in a Flow/Grid layout

## 7. Performance & Smart Polling

### Current Optimizations (keep)

- **Async subprocess execution** with per-command backoff tracking
  - 0‚Äì2 failures: always retry
  - 3‚Äì9 failures: try once every 4 poll cycles (~60s)
  - 10+ failures: disable entirely
- **Deduplication:** `m_runningCommands` set prevents overlapping processes
- **2-second timeout** per subprocess to prevent hangs
- **D-Bus property reads** (battery, NM state, BlueZ) are synchronous but fast
  (~1ms each) ‚Äî acceptable at 15s interval

### New Optimizations (implement)

| Optimization | Description |
|---|---|
| **Signal-driven updates** | Subscribe to `PropertiesChanged` on NM, BlueZ, UPower ‚Äî poll only as fallback |
| **Adaptive poll interval** | When panel is **closed**: poll every 30s (status bar only needs battery/wifi icon). When panel is **open**: poll every 5s for responsive sliders |
| **Lazy sub-page loading** | Wi-Fi network list and Bluetooth device scan only trigger when sub-page is opened |
| **MPRIS is zero-poll** | Entirely signal-driven ‚Äî no timer needed |
| **SNI is zero-poll** | Entirely signal-driven ‚Äî no timer needed |
| **Slider coalescing** | During drag, coalesce `setBrightness`/`setVolume` calls ‚Äî apply at most once per 50ms |
| **Thumbnail caching** | Cache MPRIS album art in `$XDG_CACHE_HOME/chi-shell/mpris-art/` |
| **Hardware detection at startup** | On init, detect: has battery? has backlight? has BT adapter? has accelerometer? Store as flags, hide irrelevant tiles permanently |

| `surfaceContainerHigh` | Inactive tile / slider track background |
| `primaryContainer` | Active tile / slider fill |
| `onPrimaryContainer` | Active tile icon + text |
| `onSurface` | Header time text |
| `onSurfaceVariant` | Secondary text, inactive icons |

**Dark mode toggle** in the tile grid instantly recolors the entire panel
via `ChiTheme.toggleDarkMode()`.

## 11. Future Considerations

Items **not** in scope for v0.3 but designed to be possible:

| Feature | Notes |
|---|---|

| **Live weather/info tile** | A double-wide tile showing weather or calendar next event |
| **Tile badges** | Notification count badge on tiles (e.g. "3 BT devices connected") |
| **Per-output panels** | On multi-monitor setups, Quick Settings follows the active output |
