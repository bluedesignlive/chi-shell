## 1. Overview

The Quick Settings panel is a full-screen overlay surface on the `Overlay` layer.
It provides instant access to system controls, media playback, connectivity,
and third-party app actions ÔÇö without ever opening a full Settings application.

**Design principles:**

- One-tap for toggles, expand-tap for details
- Every control must respond in under 100ms (optimistic UI)
- No synchronous process calls on the main thread ÔÇö ever
- D-Bus first, CLI fallback, sysfs for hardware reads
- Panel must work with mouse, touch, keyboard, and pen

- **Open:** Panel translates from `y: -20` to `y: 0` with `opacity: 0 ÔåÆ 1`

### B) Media Player (Now Playing)

A card that appears **only when a media player is active** via MPRIS2.

| Element          | Details                                         |
|------------------|-------------------------------------------------|
| Album art        | 48├ù48 rounded square, left side                 |
| Track title      | Single line, ellipsis overflow                   |
| Artist name      | Single line, secondary color                     |
| Player name      | Subtle label (e.g. "Spotify", "Firefox")         |
| Controls         | ÔùÇÔùÇ  ÔûÂ/ÔÅ©  ÔûÂÔûÂ ÔÇö centered row                     |
| Progress         | Thin progress bar beneath the card (optional)    |

**D-Bus source:** `org.mpris.MediaPlayer2.*` on session bus.

| Interface                          | Usage                              |
|------------------------------------|------------------------------------|
| `org.mpris.MediaPlayer2.Player`    | Metadata, PlaybackStatus, controls |
| `Properties.PropertiesChanged`     | Live updates without polling       |

**Behavior:**
- Discover players by watching bus names matching `org.mpris.MediaPlayer2.*`.
- When multiple players exist, show the one currently playing. If none are
  playing, show the most recently active.
- Tap ÔûÂ/ÔÅ© calls `PlayPause()`. Tap ÔùÇÔùÇ/ÔûÂÔûÂ calls `Previous()`/`Next()`.
- Tap the card body opens nothing (future: expand to show seek bar).
- Card hides with a 5-second delay after the player disappears from the bus.

**Performance:**
- No polling. Use `PropertiesChanged` signal for live metadata updates.
- Cache album art URL. Only reload image when `mpris:artUrl` changes.

### C) Brightness Slider

A horizontal pill-shaped slider for display brightness.

| Element         | Details                                         |
|-----------------|-------------------------------------------------|
| Track           | Full-width rounded rectangle, `surfaceContainerHigh` |
| Fill            | Left-to-right, `primaryContainer` color          |
| Icon            | `brightness_medium` pinned to right side         |
| Height          | 40px (44px hit target)                           |
| Animation       | `SpringAnimation { spring: 14.0; damping: 1.0 }`|

**Backend:** Read from `/sys/class/backlight/*/brightness` (sysfs).
Write via `brightnessctl set <value>`.

**Behavior:**
- Drag or tap anywhere on the track to set brightness.
- Value is `0.0` to `1.0`, mapped to hardware range.
- On desktops without a backlight device, hide this slider entirely.
- Optimistic UI: update fill immediately, write to hardware async.

**Performance:**
- sysfs reads are fast (file I/O). No D-Bus or process spawn needed for reads.
- Writes use `QProcess::startDetached` ÔÇö fire and forget.
- Throttle writes during drag: apply at most every 50ms (accumulate last value).

### D) Volume Slider

Identical visual style to Brightness slider.

| Element         | Details                                         |
|-----------------|-------------------------------------------------|
| Fill            | `primaryContainer` color, width = `volume / 100`|
| Icon            | Dynamic: `volume_up`, `volume_down`, `volume_mute`, `volume_off` |
| Range           | 0ÔÇô100 (integer percent)                         |

**Backend:** PipeWire/PulseAudio via `pactl`.

| Action           | Command                                        |
|------------------|------------------------------------------------|
| Read volume      | `pactl get-sink-volume @DEFAULT_SINK@`         |
| Read mute        | `pactl get-sink-mute @DEFAULT_SINK@`           |
| Set volume       | `pactl set-sink-volume @DEFAULT_SINK@ XX%`     |
| Set mute         | `pactl set-sink-mute @DEFAULT_SINK@ 1/0`       |

**Behavior:**
- Drag or tap to set volume.
- Optimistic UI: update fill immediately.
- If muted, show fill at 0% but remember actual volume for unmute.

**Performance:**
- Read commands are async with 2s timeout and backoff on failure.
- Writes are `startDetached` ÔÇö fire and forget.
- Throttle drag writes to every 50ms.

**Future:** Replace `pactl` with native PipeWire D-Bus or `libpipewire`
for zero-process-spawn volume control.

## 6. System Tray Integration (StatusNotifierItem)

Chi-shell acts as both `StatusNotifierWatcher` and `StatusNotifierHost`.

**Discovery:**
- Register `org.kde.StatusNotifierWatcher` on the session bus.
- Apps register themselves via `RegisterStatusNotifierItem(service)`.
- Chi-shell tracks active items and emits `StatusNotifierItemRegistered` /
  `StatusNotifierItemUnregistered`.

**Display:**
- Tray icons appear in the **StatusBar** (top bar), not in Quick Settings.
- If an SNI item's menu is accessed from the StatusBar, it uses `com.canonical.dbusmenu`
  to render a popup menu.

**Future:** Allow users to promote an SNI item into a Quick Settings tile.

## 7. Performance Model

### Polling Strategy

| Data Source      | Method         | Interval | Notes                            |
|------------------|----------------|----------|----------------------------------|
| Battery          | D-Bus property | 15s      | UPower ÔÇö always fast             |
| Wi-Fi state      | D-Bus property | 15s      | NM state enum                    |
| Wi-Fi SSID       | `nmcli` async  | 15s      | With `--wait 0`, 2s timeout     |
| Bluetooth        | D-Bus property | 15s      | BlueZ ÔÇö always fast              |
| Volume           | `pactl` async  | 15s      | 2s timeout, backoff on failure   |
| Mute             | `pactl` async  | 15s      | 2s timeout, backoff on failure   |
| Brightness       | sysfs read     | 15s      | File I/O ÔÇö always fast           |
| Media            | D-Bus signal   | ÔÇö        | No polling. Signal-driven.       |
| Power profile    | D-Bus property | 15s      | Single property read             |

### Async Command Rules

1. All `QProcess` calls are **async** (`QProcess::finished` signal).
2. Each command ID tracks consecutive failures.
3. After 3 failures ÔåÆ back off to once per minute.
4. After 10 failures ÔåÆ disable until next shell restart.
5. Never run two instances of the same command concurrently.
6. Every async command has a 2-second kill timer.

### Optimistic UI

All user-initiated changes (toggle, slider drag) update the QML-bound
property **immediately** before the system command executes. If the command
fails, the next poll cycle corrects the state.

### Slider Throttling

During active drag on brightness/volume sliders, writes are throttled
to one `startDetached` call per 50ms. The last value always gets applied
on release.

## 10. Configuration

User preferences are stored in `~/.config/chi-shell/quicksettings.json`.

```json
{
  "version": 1,
  "tiles": [
    { "id": "wifi",          "size": "wide",  "row": 0, "col": 0 },
    { "id": "bluetooth",     "size": "wide",  "row": 0, "col": 1 },
    { "id": "dark_mode",     "size": "wide",  "row": 1, "col": 0 },
    { "id": "dnd",           "size": "small", "row": 1, "col": 1 },
    { "id": "mute",          "size": "small", "row": 1, "col": 2 },
    { "id": "power_profile", "size": "wide",  "row": 2, "col": 0 },
    { "id": "night_light",   "size": "wide",  "row": 2, "col": 1 }
  ],
  "showMediaPlayer": true,
  "sliderOrder": ["brightness", "volume"]
}
```

**Edit mode:** Tap a pencil/edit icon in the footer to enter edit mode.
Tiles become draggable. A tray of available (hidden) tiles appears at the
bottom. Drag tiles in/out to add/remove. Drag to reorder. Tap "Done" to
save and exit edit mode.

## 11. Future Considerations

- **Third-party tile API** (`org.chi.QuickSettings.Tile` D-Bus interface)
  allowing any app to register a toggle tile with icon, label, state, and
  activate callback.
- **Resizable tiles** ÔÇö hold a tile in edit mode and resize between
  small / wide / large (full-width card with extra info).
- **Multiple panels** ÔÇö swipe left/right between custom tile groups
  (e.g. "Home", "Media", "Dev Tools").
- **XDG Desktop Portal backend** (`xdg-desktop-portal-chi`) so Flatpak
  and sandboxed apps can read color-scheme, request screenshots, and
  interact with the shell natively.
- **Native PipeWire integration** ÔÇö replace `pactl` with direct PipeWire
  D-Bus or `libpipewire` for zero-latency, zero-process-spawn audio control.
- **Power menu expansion** ÔÇö shutdown, restart, suspend, log out, lock
  screen ÔÇö triggered from the footer power button.
- **Per-app audio** ÔÇö volume sliders for individual apps
  (PipeWire stream enumeration).
